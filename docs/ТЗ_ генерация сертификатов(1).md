## **Функционал: редактор шаблона сертификатов**

Редактор шаблона сертификатов — это инструмент в админ-панели, с помощью которого администратор настраивает внешний вид сертификата и определяет, какие данные и в каком месте будут автоматически подставляться при генерации.

### **Что должен уметь редактор**

#### **1\. Загрузка основы сертификата**

Администратор должен иметь возможность загрузить основу сертификата:

* PNG  
* JPG  
* PDF

Это будет фон, поверх которого система размещает текстовые данные.

#### **2\. Настройка поля имени**

Администратор должен иметь возможность настроить поле, в которое будет автоматически подставляться имя участника из таблицы.

Для этого нужно уметь задавать:

* позицию по горизонтали и вертикали  
* ширину области  
* выравнивание текста  
* размер шрифта  
* тип шрифта  
* цвет текста  
* жирность  
* межстрочный интервал, если потребуется  
* автоматическое уменьшение шрифта для длинных имен

#### **3\. Добавление других динамических полей**

Помимо имени, редактор должен позволять добавлять и другие переменные поля, например:

* {{full\_name}}  
* {{track\_name}}  
* {{certificate\_type}}  
* {{issue\_date}}  
* {{certificate\_id}}

Каждое поле должно настраиваться отдельно по тем же параметрам:

* позиция  
* размер  
* шрифт  
* цвет  
* выравнивание

#### **4\. Добавление статичных надписей**

Администратор должен иметь возможность добавлять на сертификат не только переменные, но и обычный текст, например:

* название хакатона  
* подпись  
* название трека  
* описание сертификата

Такой текст тоже должен редактироваться по стилю и расположению.

#### **5\. Визуальное размещение элементов**

Редактор должен работать визуально, чтобы администратор видел шаблон сертификата и мог размещать элементы прямо на макете.

Желательно:

* drag-and-drop размещение  
* перемещение элементов мышкой  
* отображение координат  
* возможность вручную задавать точные координаты

#### **6\. Превью результата**

Редактор должен показывать превью итогового сертификата с тестовыми данными до публикации.

Например:

* вместо {{full\_name}} показывать тестовое имя  
* вместо {{track\_name}} показывать тестовый трек

Это нужно, чтобы администратор сразу видел, как будет выглядеть готовый сертификат.

#### **7\. Сохранение шаблона**

После настройки администратор должен иметь возможность:

* сохранить шаблон  
* обновить существующий шаблон  
* создать новый шаблон на основе текущего  
* выбрать шаблон как активный

#### **8\. Проверка корректности шаблона**

Перед сохранением система должна проверять:

* что обязательное поле имени добавлено  
* что текст не выходит за пределы сертификата  
* что выбран допустимый шрифт и размер  
* что шаблон можно использовать для генерации

### **Что желательно добавить**

Для удобства можно предусмотреть:

* дублирование элементов  
* удаление элементов  
* слои  
* блокировку элементов от случайного сдвига  
* набор безопасных шрифтов  
* поддержку нескольких шаблонов под разные треки или типы сертификатов

### **Результат работы редактора**

На выходе редактор должен сохранять конфигурацию шаблона, по которой система сможет автоматически генерировать сертификаты для пользователей на основе данных из загруженной таблицы.
